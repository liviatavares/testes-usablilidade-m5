As 4 telas acima representam um fluxo de usuário para conseguir gerar o relatório de uma resposta a uma ocorrência já existente. A partir da tela inicial (figura 01), o fluxo é o seguinte:

Tela inicial -> Criar ocorrência -> Selecionar os recursos -> Gerar ocorrência -> Tela inicial -> Gerar resposta -> Selecionar as UPS e gerar respostas múltiplas -> Selecionar uma resposta -> Finalizar e despachar -> Gerar relatório -> Tela de respostas ativas -> Clicar em "PDF" para salvar o relatório da resposta em pdf.

Para fins de simplificação, espera-se que a ocorrência já esteja criada, bem como a resposta a ela, de modo a testar a funcionalidade de gerar relatório, então o fluxo simplifica-se para:

Tela inicial -> Gerar relatório -> Tela de respostas ativas -> Clicar em "PDF" para salvar o relatório da resposta em pdf.

## 2. Tipo de teste

O teste é um teste de tarefa, pois o objetivo é verificar se o usuário consegue completar um fluxo específico.

O objetivo é responder à seguinte pergunta: “o usuário consegue executar a tarefa de gerar relatório de uma resposta de ocorrência com eficiência?”

## 3. Conjunto de perguntas

1. Você precisa gerar um relatório de uma resposta já existente. Por onde começaria nessa tela?
2. O que te faz escolher “Gerar relatório” em vez de outras opções como “Acessar respostas” ou “Gerar resposta”?
3. Ao chegar nessa lista, como você identifica qual resposta usar para gerar o relatório?
4. O botão “PDF” atende ao que você espera para gerar o relatório? O que você acha que vai acontecer ao clicar nele?
5. Essa tela de impressão/salvar como PDF faz sentido para você? Você esperava esse comportamento ou algo diferente dentro do sistema?

## 4. Objetivo do teste

Identificar dificuldades na navegação e compreensão do fluxo de geração de relatório, especialmente na descoberta da funcionalidade e na interpretação da ação de exportação em PDF.

## 5. Ação ou entendimento esperado

O usuário deve conseguir gerar e salvar o relatório em PDF a partir de uma resposta existente, sem auxílio, entendendo corretamente onde iniciar o fluxo e como executar a ação final.
